<Update label="2026-05-03" description="May 2026" tags={["Fixes"]}>
## Fixes

**More reliable AI auto-moderation** — [AI auto-moderation](/features/moderation) now parses model responses more robustly, so messages are scored and acted on correctly even when the model wraps its output in extra text or nested formatting. Fewer skipped messages, fewer "no decision" outcomes.

**Stronger link filter and phishing detection** — The link filter now extracts URLs more accurately from messages, catching more obfuscated and disguised links. Phishing detection has been hardened against common evasion tricks (extra punctuation, mixed casing, embedded characters), so suspicious links are flagged even when posters try to sneak them past.
</Update>

<Update label="2026-05-02" description="May 2026" tags={["Improvements", "Fixes"]}>
## Improvements

**Stronger phishing and link filtering** — The link filter now catches malicious URLs hidden inside markdown links (`[click here](https://...)`), angle-bracket links (`<https://...>`), and links wedged against punctuation. Phishing keywords in URL query strings (for example, `?gift=nitro` on a `.xyz` domain) are also detected. See [moderation](/features/moderation) for the full set of safeguards.

**Better reliability monitoring** — Volvox.Bot now tracks dashboard errors and performance issues in the background so problems get spotted and fixed faster. Sensitive data — Discord IDs, message content, tokens, query params, headers, and request bodies — is scrubbed before anything leaves your browser, keeping your server's data private by default.

## Fixes

**More reliable AI auto-moderation** — [AI auto-moderation](/features/moderation) now parses AI provider responses correctly even when they include extra text or nested braces, and treats unparseable responses as "needs review" instead of letting them through. Fewer missed flags, fewer surprises.
</Update>

<Update label="2026-05-08" description="May 2026" tags={["Improvements"]}>
## Improvements

**Global admin access for ops pages** — The dashboard's **Logs** and **Performance** pages are now restricted to global admins (configured via `BOT_OWNER_IDS`). Server admins and moderators continue to manage their servers as usual, while infrastructure-level views are reserved for operators. See [Permissions](/configuration/permissions) for the full role breakdown.

**Cleaner access levels** — Server access roles are now consistently `admin`, `moderator`, or `viewer` across the dashboard and API. Global admin status is surfaced separately, so sidebar navigation and protected pages reflect the correct permissions for every user.
</Update>

<Update label="2026-05-01" description="May 2026" tags={["Features", "Improvements", "Fixes"]}>
## New features

**Branded 404 page** — Visiting a missing page on [volvox.bot](https://volvox.bot) now shows a polished, on-brand 404 with quick links back into the [dashboard](/dashboard) and docs, instead of the generic browser error.

## Improvements

**Audit trail for AI triage moderation** — Moderation actions taken by [AI automation](/features/ai-conversations) now write to the audit log, so mods can see exactly which AI decisions led to deletes, warnings, timeouts, kicks, or bans. Triage budget alerts are also throttled to avoid noisy duplicate notifications when usage limits are hit.

## Fixes

**Dashboard activity chart dates** — The activity chart on the [dashboard](/dashboard) overview now plots dates correctly, so daily message and command counts line up with the right day in your server's timezone.

**Documentation link in profile dropdown** — The "Documentation" link in the profile dropdown menu works again across the [dashboard](/dashboard), opening the docs in a new tab as expected.
</Update>

<Update label="2026-04-29" description="April 2026" tags={["Features", "Improvements", "Fixes"]}>
## New features

**AI Auto-Moderation** — Active Sentry now scores incoming messages with AI across toxicity, spam, harassment, hate speech, sexual content, violence, and self-harm. Choose a detection model, set per-category sensitivity thresholds, and configure response actions (ignore, delete, flag, warn, timeout, kick, or ban) — including multiple actions per violation. AI auto-mod actions are logged alongside classic moderation cases. Configure under **Config → Moderation & Safety → Content Safety** in the [dashboard](/dashboard). See the [moderation docs](/features/moderation) for details.

**Configurable model for `/tldr`** — You can now choose which provider/model the [TL;DR command](/features/tldr) uses to generate summaries, right from the dashboard. MiniMax M2.7 is the new default.

**MiniMax provider** — MiniMax is now a first-class AI provider, with models like **MiniMax M2.7** and **MiniMax M2.7 Highspeed** available across [AI automation](/features/ai-conversations) and [TL;DR](/features/tldr).

**Welcome panel publishing from the dashboard** — Publish, update, and remove your welcome and rules panels directly from the [dashboard](/dashboard) under Onboarding & Growth. Channel changes automatically clean up stale panel messages, and publish actions show clear success/failure feedback.

**Dedicated role menu channel** — Onboarding & Growth now exposes a separate **Role Menu Channel** picker, so your self-serve role menu can live in its own channel apart from rules and welcome.

## Improvements

**Cleaner model selectors** — Dashboard provider/model pickers now hide models that aren't available to your server, so you only see options you can actually use. Saved selections with `provider:model` IDs (including those containing colons) are preserved correctly.

**Streamlined dashboard overview** — The AI cost tile has been removed from the dashboard overview to keep the page focused on activity and health metrics.

## Fixes

**Auto-mod flag embeds** — Flag embeds now show the global actions that ran for an auto-mod violation, so moderators can see exactly what was applied at a glance.

**Welcome publish reliability** — Several edge cases in welcome panel publishing have been resolved: split payloads, status mismatches, persistence gaps, channel-change cleanup, and rate-limited publish routes are all handled more gracefully.

**AI auto-mod stability** — Hardened prompt handling, action reporting, and case/flag log writes so AI auto-moderation behaves predictably under unusual server conditions.
</Update>

<Update label="2026-04-22" description="April 2026" tags={["Features"]}>
## New features

**Level-up action builder** — You can now configure automated actions that trigger when members reach specific XP levels. From the dashboard, set up actions like granting or removing roles, sending DMs, or posting announcements in a channel. You can also award bonus XP, add reactions, set nickname prefixes or suffixes, and fire webhooks. Each action supports Discord markdown messages, rich embeds, and template variables like `{{username}}` and `{{level}}`. Assign multiple actions per level to create rich, customized level-up experiences for your server.
</Update>

<Update label="2026-04-21" description="April 2026" tags={["Features", "Improvements", "Fixes"]}>
## New features

**Role-based triage filtering** — You can now control which users the AI responds to by role. Set allowed roles (whitelist) or excluded roles (blacklist) in your AI Automation settings. Bot, webhook, and system messages are automatically filtered out.

**Configurable level-up DMs** — You can now set custom DM messages sent to members when they level up. Supports per-level message overrides, a live Discord-style preview, and built-in rate limiting to prevent spam.

**Discord markdown editor** — A new rich-text editor for Discord-formatted messages is available in the dashboard. Includes a formatting toolbar, variable inserter, split raw/preview mode, character counter, and keyboard shortcuts.

**Visual embed builder** — Create Discord embeds visually in the dashboard with a color picker, field management, thumbnail selector, and a live embed preview.

**Dashboard permission controls** — The dashboard now enforces role-based access. Viewers get read-only access, moderators can manage moderation features, and admins have full control.

**Server-scoped log viewer** — The log viewer now shows logs scoped to your current server. A new channel filter lets you narrow the live log stream to specific Discord channels.

## Improvements

**Landing page redesign** — We completely rebuilt the landing page with a new visual style, interactive feature previews in a bento-box grid, animated stats, and a redesigned pricing section.

**Dashboard polish** — Row hover highlighting on tables, animated KPI counters, visible heatmap tooltips, clickable member rows, corrected XP progress bars, and a live commands-served counter. Thin scrollbars on all inputs and dropdowns for a cleaner look.

**Identity resolution** — Conversation replays now show real Discord profile icons. Mentions are resolved to actual usernames, and audit logs display rich user identity (display name + username).

**Improved AI reliability** — The AI triage system now includes retry logic, timeout handling, and per-role budget support. You can configure separate models for classification vs. responding to optimize costs.

**Mobile experience** — The settings save toolbar, header, and sidebar are now fully responsive on smaller screens. Reduced-motion accessibility support is available across all animated components.

**OpenGraph previews** — Sharing Volvox.Bot links now shows rich previews on social media, Discord, and Slack. A sitemap and robots.txt have been added for better search engine discoverability.

**Branding update** — We standardized all visible branding to "Volvox.Bot" across the product.

## Fixes

**More reliable bot responses** — Command responses across reminders, moderation, reaction roles, and other features now handle failures gracefully (e.g., blocked DMs or deleted channels) instead of silently failing.

**Bot owners setting removed** — We removed the "Bot Owners" editor from dashboard Permissions settings. Bot ownership is now managed via environment configuration only.
</Update>
