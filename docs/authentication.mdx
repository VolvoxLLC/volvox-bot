# Authentication

The API supports two authentication methods:

## OAuth2 Session (Bearer JWT)

The web dashboard uses Discord OAuth2. When you log in at [volvox.bot](https://volvox.bot):

1. Click **Login with Discord**
2. Discord redirects back with an authorization code
3. The server exchanges the code for tokens and returns a signed JWT
4. Subsequent requests include `Authorization: Bearer <jwt>`

This is the recommended method for integrations that act on behalf of a user.

## API Key

For server-to-server integrations, generate an API key from the dashboard:

1. Go to **Settings → API Keys**
2. Click **Generate New Key**
3. Include the key in the `x-api-secret` header:

```bash
curl -H "x-api-secret: YOUR_KEY" \
  https://volvox.bot/api/v1/health
```

<Callout type="warn">
API keys are shown once at creation. Store them securely — you can't view them again.
</Callout>

## Public Endpoints

These endpoints require no authentication:

| Endpoint | Description |
|----------|-------------|
| `GET /health` | Basic health check |
| `GET /community/{guildId}/leaderboard` | Public leaderboard |
| `GET /community/{guildId}/showcases` | Public showcases |
| `GET /community/{guildId}/stats` | Public server stats |
| `GET /community/{guildId}/profile/{userId}` | Public user profile |
