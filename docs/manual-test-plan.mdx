# Manual Test Plan

Use this page as the canonical manual QA checklist for release candidates and broad regression passes.

## Scope

- Discord bot runtime and command behavior
- Dashboard/web app UX and settings flows
- API integration and error handling
- Data persistence and per-guild isolation
- Permissions, moderation safety, and auditability
- Accessibility, resilience, and operational recovery

## Execution model

- Run targeted suites for normal PR validation.
- Run the full suite set (A–O) for release candidates.
- Capture evidence for failures (screenshots, logs, timestamps, guild/persona context).

- [`docs/wiki-pages/Manual-Test-Plan.md`](https://github.com/VolvoxLLC/volvox-bot/blob/main/docs/wiki-pages/Manual-Test-Plan.md)
