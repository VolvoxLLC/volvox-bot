# Web Dashboard

The Volvox Bot admin dashboard (`web/`) is a Next.js application that provides a browser-based interface for managing your Discord server's bot configuration, moderation actions, and analytics.

## Getting Started

### Requirements

- Node.js 22+
- pnpm
- A running Volvox Bot instance with `BOT_API_PORT` exposed (default: 3001)
- Discord OAuth2 application credentials

### Setup

```bash
cd web
cp .env.example .env.local
# Fill in your Discord OAuth2 credentials and bot API URL
pnpm install --legacy-peer-deps
pnpm dev
```

The dashboard will be available at `http://localhost:3000`.

> **Note:** `--legacy-peer-deps` is required due to NextAuth v4 + Next.js 16 peer dependency constraints.

### Environment Variables

| Variable | Description |
|----------|-------------|
| `NEXTAUTH_URL` | Canonical URL of the dashboard (e.g. `http://localhost:3000`) |
| `NEXTAUTH_SECRET` | Random secret for JWT encryption. Generate with `openssl rand -base64 48` |
| `DISCORD_CLIENT_ID` | Discord OAuth2 application client ID |
| `DISCORD_CLIENT_SECRET` | Discord OAuth2 application client secret |
| `BOT_API_URL` | URL of the bot's REST API (e.g. `http://localhost:3001/api/v1`) |
| `BOT_API_SECRET` | Shared secret matching `BOT_API_SECRET` in the bot's `.env` |

## Signing In

1. Navigate to the dashboard URL
2. Click **Sign in with Discord**
3. Authorize the OAuth2 app â€” you'll only see servers where you and the bot are both present
4. Select your server from the server picker

Sessions persist via NextAuth.js JWTs. Discord tokens are refreshed automatically when they expire.

The dashboard supports **dark mode** and **light mode** with system preference detection. Use the theme toggle in the top-right corner to switch.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Can't see my server | Ensure bot is in the server. Check `BOT_API_URL` and `BOT_API_SECRET` in `.env.local`. |
| "Session expired" errors | Verify `NEXTAUTH_SECRET` is set and consistent (don't change it between restarts). |
| Config saves fail | Check `BOT_API_URL` points to the running bot API. Verify the config key is in `SAFE_CONFIG_KEYS`. |
| Role/channel dropdowns empty | Check bot has `View Channels`, `Read Message History`, and `Manage Roles` permissions in the server. |
| AI feedback page shows no data | Set `ai.feedback.enabled: true` in config and ensure users have started reacting to bot messages. |
