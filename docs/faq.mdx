# Frequently Asked Questions

<Accordion>
<AccordionItem title="Is Volvox.Bot free?">

Volvox.Bot offers a free tier with core features. Premium plans unlock advanced features like AI conversations, detailed analytics, and priority support. See the [pricing page](https://volvox.bot#pricing) for details.

</AccordionItem>

<AccordionItem title="Do I need to host anything?">

No. Volvox.Bot is fully hosted. Add it to your Discord server and everything runs on our infrastructure. No servers to manage, no updates to install.

</AccordionItem>

<AccordionItem title="Is my data secure?">

Yes. All data is encrypted in transit (TLS 1.3) and at rest. We never store your Discord password — authentication is via OAuth2. See the [Security page](/security) for full details.

</AccordionItem>

<AccordionItem title="What Discord permissions does the bot need?">

The bot requests permissions it needs to function — reading messages, managing channels for tickets, assigning roles, and moderation tools. See the [Permissions page](/configuration/permissions) for the full list.

</AccordionItem>

<AccordionItem title="Can I use it in multiple servers?">

Yes. Each server has its own independent configuration. Add the bot to as many servers as you need.

</AccordionItem>

<AccordionItem title="How do I get support?">

Join our [Discord community](https://discord.gg/8ahXACdamN) for help, or open an issue on [GitHub](https://github.com/VolvoxLLC/volvox-bot/issues). See the [Help page](/help) for all support channels.

</AccordionItem>

<AccordionItem title="What happens if the bot goes offline?">

Volvox.Bot is designed for 99.99% uptime with automatic failover. If the bot does go offline, it recovers automatically and processes any missed events.

</AccordionItem>

<AccordionItem title="Can I export my data?">

Yes. Contact support for a data export, or use the dashboard to download your data.

</AccordionItem>

<AccordionItem title="Does it support slash commands?">

Yes. Volvox.Bot uses slash commands for all Discord interactions. Type `/` in any channel where the bot is present to see available commands.

</AccordionItem>
</Accordion>
