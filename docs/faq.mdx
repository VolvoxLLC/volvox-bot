# Frequently Asked Questions

<AccordionGroup>
<Accordion title="Is Volvox.Bot free?">

Volvox.Bot offers a free tier with core features. Premium plans unlock advanced features like AI conversations, detailed analytics, and priority support. See the [pricing page](https://volvox.bot#pricing) for details.

</Accordion>

<Accordion title="Do I need to host anything?">

No. Volvox.Bot is fully hosted. Add it to your Discord server and everything runs on our infrastructure. No servers to manage, no updates to install.

</Accordion>

<Accordion title="Is my data secure?">

Yes. Connections are encrypted and data is stored securely. We never store your Discord password — authentication is via OAuth2. See the [Security page](/security) for full details.

</Accordion>

<Accordion title="What Discord permissions does the bot need?">

The bot requests permissions it needs to function — viewing channels, managing channels for tickets, assigning roles, and moderation tools. See the [Permissions page](/configuration/permissions) for the full list.

</Accordion>

<Accordion title="Can I use it in multiple servers?">

Yes. Each server has its own independent configuration. Add the bot to as many servers as you need.

</Accordion>

<Accordion title="How do I get support?">

Join our [Discord community](https://discord.gg/8ahXACdamN) for help, or open an issue on [GitHub](https://github.com/VolvoxLLC/volvox-bot/issues). See the [Help page](/help) for all support channels.

</Accordion>

<Accordion title="What happens if the bot goes offline?">

Volvox.Bot is designed for high availability with automatic recovery. If the bot does go offline, it restarts automatically and resumes normal operation.

</Accordion>

<Accordion title="Can I export my data?">

Yes. Contact support for a data export, or use the dashboard to download your data.

</Accordion>

<Accordion title="Does it support slash commands?">

Yes. Volvox.Bot uses slash commands for all Discord interactions. Type `/` in any channel where the bot is present to see available commands.

</Accordion>
</AccordionGroup>
